Absolute postion will be positioned relative to the nearest po sition of ancestor. One exception is that the parent is positoned static. Using position:absolute will exclude that element from the normal flow of the page. We can use top, left, right, and it will also block the existing elemnt if there is any. Also, the parent element matters in this case. If we don't set the position of container, it is default static, but if we set it to relative or absolute, the green box will be on the bottom of that element.

This is because if the parent element is static, the absolute position attribute will attach to the body element except for the parent element.

CSS also has a z-index value, so the higher z-index value element will win and be displayed. But to do this, we have to have position to not static, something like relative.

Also, if the element has postion absolute and if it does not have height and width, it appears like in-line elements. That is, it will have full-width content.

Fixed position is displayed as box 4. It will remain there regardless of other elements, and stay there depending the viewport. It will be used such as navigations.

(Internet explorer does not support sticky)
Sticky postion will act something like fixed, but also has to have top attribute, to stick to the top, until the parent container is over.
